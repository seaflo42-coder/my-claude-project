# 🎯 PC↔모바일 자동화 플러그인 - 초보자 가이드

코딩을 잘 모르셔도 따라하실 수 있도록 단계별로 설명드립니다.

---

## 📋 사전 준비물

1. **Node.js** (필수)
2. **Figma Desktop 앱** (필수 - 웹 버전에서는 개발 플러그인 로드 불가)

---

## Step 1: Node.js 설치하기

> Node.js는 플러그인 코드를 변환(빌드)하는 데 필요한 프로그램입니다.

### Windows
1. https://nodejs.org 접속
2. **LTS** 버전 다운로드 (왼쪽 초록 버튼)
3. 다운로드된 파일 실행 → 계속 "Next" 클릭 → "Install"

### Mac
1. https://nodejs.org 접속
2. **LTS** 버전 다운로드
3. 다운로드된 .pkg 파일 실행 → "계속" 클릭 → 설치 완료

### 설치 확인
터미널(또는 명령 프롬프트)에서:
```
node -v
```
`v18.x.x` 같은 버전 번호가 나오면 성공!

---

## Step 2: 터미널 열기

### Windows
- `Win + R` → `cmd` 입력 → Enter
- 또는 시작 메뉴에서 "명령 프롬프트" 검색

### Mac
- `Cmd + Space` → "터미널" 검색 → Enter
- 또는 응용 프로그램 → 유틸리티 → 터미널

---

## Step 3: 플러그인 폴더로 이동하기

터미널에서 플러그인 폴더로 이동해야 합니다.

### 방법 1: 직접 경로 입력
```bash
cd /플러그인이/있는/경로/figma-plugin
```

### 방법 2: 폴더 드래그 (더 쉬움!)

**Mac:**
1. 터미널에 `cd ` 입력 (cd 뒤에 공백!)
2. Finder에서 `figma-plugin` 폴더를 터미널 창으로 드래그
3. Enter

**Windows:**
1. `figma-plugin` 폴더를 파일 탐색기에서 열기
2. 주소창 클릭 → `cmd` 입력 → Enter
3. 해당 폴더에서 바로 명령 프롬프트가 열림

---

## Step 4: 의존성 설치하기 (최초 1회만)

터미널에서 다음 명령어 입력:

```bash
npm install
```

⏳ 잠시 기다리면 필요한 파일들이 자동으로 다운로드됩니다.

**성공하면 이런 메시지가 나옵니다:**
```
added 123 packages in 10s
```

**에러가 나면:**
- Node.js가 제대로 설치되었는지 확인
- 터미널을 껐다가 다시 열고 시도

---

## Step 5: 빌드하기

터미널에서 다음 명령어 입력:

```bash
npm run build
```

**성공하면 이런 메시지가 나옵니다:**
```
dist/code.js  xxx kb
```

✅ `dist/code.js` 파일이 생성되면 빌드 완료!

---

## Step 6: Figma Desktop에서 플러그인 불러오기

> ⚠️ 반드시 **Figma Desktop 앱**을 사용하세요! (웹 버전 안 됨)

### 6-1. Figma Desktop 다운로드
아직 없다면: https://www.figma.com/downloads/

### 6-2. 플러그인 불러오기

1. Figma Desktop 실행
2. 아무 파일이나 열기 (또는 새 파일 생성)
3. 상단 메뉴 → **Plugins** → **Development** → **Import plugin from manifest...**

   ![메뉴 위치](https://help.figma.com/hc/article_attachments/4404878234135)

4. 파일 선택 창에서 `figma-plugin` 폴더 안의 **manifest.json** 파일 선택
5. "Open" 클릭

### 6-3. 플러그인 실행하기

1. 상단 메뉴 → **Plugins** → **Development** → **PC↔모바일 상세페이지 자동화**
2. 또는 단축키: `Cmd/Ctrl + /` → "PC모바일" 검색 → Enter

---

## Step 7: 컴포넌트 이름 규칙 적용하기

플러그인이 PC/모바일 모듈을 인식하려면 컴포넌트 이름을 수정해야 합니다.

### 현재 이름 → 변경할 이름

| 기존 | PC 버전 | 모바일 버전 |
|------|---------|-------------|
| /usp_type1 | /pc_usp_type1 | /mo_usp_type1 |
| /기본정보_type1 | /pc_기본정보_type1 | /mo_기본정보_type1 |
| /네비게이션_type1 | /pc_네비게이션_type1 | /mo_네비게이션_type1 |

### Figma에서 이름 변경하기
1. 레이어 패널에서 컴포넌트 더블클릭
2. 이름 앞에 `pc_` 또는 `mo_` 추가

---

## Step 8: 플러그인 사용하기

### 🔍 모듈 스캔
1. 플러그인 실행
2. **모듈 스캔** 탭 선택
3. **모듈 스캔 시작** 버튼 클릭
4. PC/모바일 모듈이 자동으로 표시됨

### 🔗 매핑 설정
1. PC 모듈과 대응되는 모바일 모듈 선택
2. **매핑 저장** 클릭

### 📄 페이지 생성
1. **페이지 생성** 탭 이동
2. 포함할 모듈 체크 ✓
3. 드래그해서 순서 조정
4. 페이지 이름 입력
5. **페이지 생성** 클릭!

### 🔄 콘텐츠 동기화
1. Figma에서 동기화할 요소 선택
2. **콘텐츠 동기화** 탭 이동
3. **PC → 모바일** 또는 **모바일 → PC** 클릭

## ❓ 자주 묻는 질문

### Q: npm install 에서 에러가 나요
**A:**
- Node.js 설치 후 터미널을 새로 열어주세요
- Mac에서 권한 에러시: `sudo npm install` 입력

### Q: npm run build 에서 에러가 나요
**A:**
- `npm install`을 먼저 실행했는지 확인
- `figma-plugin` 폴더 안에서 실행하고 있는지 확인

### Q: Figma에서 플러그인이 안 보여요
**A:**
- Figma **Desktop 앱**인지 확인 (웹 버전 X)
- Plugins → **Development** 메뉴에서 찾기

### Q: 모듈이 스캔되지 않아요
**A:**
- 컴포넌트 이름 앞에 `pc_` 또는 `mo_` 있는지 확인
- 컴포넌트가 **Component** 또는 **Component Set**인지 확인

### Q: 코드를 수정했는데 반영이 안 돼요
**A:**
- 코드 수정 후 `npm run build` 다시 실행
- Figma에서 플러그인 닫았다가 다시 실행
